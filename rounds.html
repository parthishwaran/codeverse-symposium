Competition Rounds

Detailed breakdown of all technical challenges

Round 1 Round 2 Round 3 Round 4

Round 1: Speed Code Sprint

10:00 AM - 11:15 AM (75 mins) Max 35 points

Description

Teams solve coding problems quickly and accurately. Question papers will contain problem statements with expected syntax and outputs.

Easy

Basic programming concepts (string manipulation, simple algorithms)

Medium

Array operations and intermediate data structure problems

Hard

Graph algorithms and complex logical problems

Scoring

  • 10 points per correct solution
  • +5 bonus points for top 5 fastest teams

Rules

  • Languages: Python, Java, C++, C
  • Submit executable code files
  • No internet access
  • No external communication
  • Language specification sheets allowed

Round 2: Algorithm Design Challenge

11:30 AM - 1:00 PM (90 mins) Max 30 points

Description

Teams solve one complex problem involving algorithm design and implementation.

Example problem types:

  • Shortest path algorithms
  • Scheduling optimization
  • Text pre-processing

Scoring

  • 10 points Correctness (passes test cases)
  • 10 points Algorithm efficiency
  • 10 points Clarity of explanation

Rules

  • Document algorithm design (pseudocode/flowchart)
  • 5-7 minute presentation to judges
  • No external resources

Round 3: Technical Connection

2:00 PM - 2:30 PM (30 mins) Max 60 points

Description

A fun, fast-paced technical quiz with visual challenges.

Example questions:

  • Identify programming languages from code snippets
  • Name tech personalities from photos
  • Solve visual logic puzzles
  • Recognize algorithms from diagrams

Scoring

  • 3 points per correct answer
  • 20 questions total

Rules

  • No electronic devices
  • No team discussions
  • Write answers on provided sheets

Round 4: System Design Showdown

2:35 PM - 4:20 PM (1h 45m) Max 70 points

Description

Teams design the architecture for a complex system or application.

Example challenges:

  • Ride-sharing app like Uber
  • Real-time collaborative code editor
  • Food delivery tracking system
  • AI-based exam proctoring system

Scoring

  • 40 points Design feasibility
  • 20 points Scalability
  • 10 points Presentation clarity

Rules

  • Use provided tools (whiteboards/digital)
  • 5 minute presentation to judges
  • Focus on design, not implementation

Winner Selection

Total Points

Tie-Breaking Rules

  1. Higher score in Round 4 (System Design)
  2. Higher score in Round 2 (Algorithm Design)
  3. Earlier submission in Round 1 (Speed Code)
